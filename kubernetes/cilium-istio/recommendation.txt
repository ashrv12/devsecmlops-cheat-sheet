Deploying Istio in Ambient Mode (which uses ztunnel for sidecar-less L4 mTLS) alongside Cilium (for eBPF-based networking and policy) is a brilliant, high-performance combination, but they will fight each other if not configured perfectly.

Because Istio Ambient intercepts traffic at the node level using iptables/ztunnel, and Cilium operates down in the kernel using eBPF, you must configure Cilium to step back slightly.

When deploying Cilium via your ArgoCD Helm charts, ensure these values are set:

cni.exclusive: false

socketLB.hostNamespaceOnly: true

kubeProxyReplacement: false (Highly recommended by Cilium when running Istio Ambient to avoid split-brain routing issues).
